# Sprint_9

Данный проект содержит набор автоматизированных UI-тестов для сервиса "Продуктовый помощник". Тесты написаны на
**Python** c применением **Pytest** и **Selenium**.
Проект реализован с использованием паттерна **Page Object Model**:

- locators/ - описание локаторов страниц
- pages/ - описание методов страниц
- tests/ - сами тестовые методы
- utils/ - вспомогательные методы для генерации данных
- conftest.py - фикстуры для тестов
- data.py - тестовые данные
- urls.py - URL ресурсы

#### CI/CD & Docker Infrastructure

Проект полностью автоматизирован для стабильного запуска и мониторинга тестов:

- **Docker & Docker Compose**: вся тестовая среда (Python + Selenoid + Браузеры) запускается в контейнерах. Это
  гарантирует, что тесты пройдут одинаково и на MacBook, и в облаке GitHub.
- **GitHub Actions Pipeline**: настроен CI-пайплайн (ci.yml), который при каждом пуше сам поднимает окружение, прогоняет
  тесты и собирает результаты.
- **Автоматические отчеты**: после каждого запуска генерируется Allure Report и автоматически публикуется на GitHub
  Page.

📊 **[Посмотреть последний отчет по тестам](https://TurusovaXenia.github.io/Sprint_9/)**

### Запуск тестов

1. **Локально через Docker**:  
   убедитесь, что Docker запущен и выполните команду в корне проекта:  
   _docker compose up --build_
2. **В облаке через GitHub Actions**:  
   выполните команду _git push_ в ветку _main_.   
   Пайплайн запустится автоматически, а отчет опубликуется на GitHub Pages.
